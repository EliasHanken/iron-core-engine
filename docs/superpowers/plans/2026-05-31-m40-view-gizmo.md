# M40 — ImViewGuizmo drop-in (view-nav widget) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Blender / Godot / Unreal-style 3D view-nav widget to the sandbox editor — an axis-handle cube in the top-right corner (six axis-snap clicks + drag-orbit), plus an "Iso" button below it that snaps the `FreeFlyCamera` to a stock 3/4 isometric pose.

**Architecture:** Vendor the single-header [Ka1serM/ImViewGuizmo](https://github.com/Ka1serM/ImViewGuizmo) (MIT) under `third_party/imviewguizmo/`. Macro-substitute the library's math types to our `iron::Vec3` / `iron::Quat` so we don't pull in GLM. The new editor-only module `engine/editor/ViewGizmo.{h,cpp}` owns the library's `IMPLEMENTATION` translation unit and a thin per-frame adapter that converts `FreeFlyCamera`'s yaw/pitch ↔ quat. The sandbox adds one call per frame.

**Tech Stack:** C++23 (MSVC `/std:c++latest`), CMake (no presets — build dir `build-vk`), Dear ImGui (already linked into `ironcore_editor`), `iron::Vec3` + `iron::Quat` (engine math). No new external dependencies beyond the vendored single-header library. Reference spec: `docs/superpowers/specs/2026-05-31-m40-view-gizmo-design.md`.

**Verification model:** Phase A is pure-logic TDD. Phase B is build-only (the header should `#include`-clean in a probe TU). Phase C extends to a unit test for `setIsometricView` (the dispatch / rendering itself stays visual-gated). Phase D adds the visual gate.

- [ ] **Step 4: Implement `quatLookAt` in `Quaternion.cpp`**

`engine/math/Quaternion.cpp` does not yet exist (everything in `Quaternion.h` is inline). Create it now with the new helper. The implementation builds an orthonormal basis with `forward` = unit vector from eye to target, picks a stable `right` and `up`, then derives the quaternion from the resulting 3×3 rotation matrix (Shepperd's method — pick the largest diagonal entry to avoid numerical issues).

Create `engine/math/Quaternion.cpp`:

```cpp
#include "math/Quaternion.h"

#include <cmath>

namespace iron {

namespace {

// Build a quaternion from a 3x3 orthonormal rotation expressed as three
// column vectors (xAxis, yAxis, zAxis). Shepperd's method — pick the
// branch whose trace term is the largest positive value so the square
// root never operates on a near-zero number.
Quat quatFromBasis(Vec3 xAxis, Vec3 yAxis, Vec3 zAxis) {
    const float m00 = xAxis.x, m01 = yAxis.x, m02 = zAxis.x;
    const float m10 = xAxis.y, m11 = yAxis.y, m12 = zAxis.y;
    const float m20 = xAxis.z, m21 = yAxis.z, m22 = zAxis.z;

    const float trace = m00 + m11 + m22;
    Quat q;
    if (trace > 0.0f) {
        const float s = std::sqrt(trace + 1.0f) * 2.0f;  // s = 4 * q.w
        q.w = 0.25f * s;
        q.x = (m21 - m12) / s;
        q.y = (m02 - m20) / s;
        q.z = (m10 - m01) / s;
    } else if (m00 > m11 && m00 > m22) {
        const float s = std::sqrt(1.0f + m00 - m11 - m22) * 2.0f;  // s = 4 * q.x
        q.w = (m21 - m12) / s;
        q.x = 0.25f * s;
        q.y = (m01 + m10) / s;
        q.z = (m02 + m20) / s;
    } else if (m11 > m22) {
        const float s = std::sqrt(1.0f + m11 - m00 - m22) * 2.0f;  // s = 4 * q.y
        q.w = (m02 - m20) / s;
        q.x = (m01 + m10) / s;
        q.y = 0.25f * s;
        q.z = (m12 + m21) / s;
    } else {
        const float s = std::sqrt(1.0f + m22 - m00 - m11) * 2.0f;  // s = 4 * q.z
        q.w = (m10 - m01) / s;
        q.x = (m02 + m20) / s;
        q.y = (m12 + m21) / s;
        q.z = 0.25f * s;
    }
    return q.normalized();
}

}  // namespace

Quat quatLookAt(Vec3 eye, Vec3 target, Vec3 up) {
    // Right-handed camera basis. The camera looks down its local -Z, so we
    // store the forward direction as `zAxis = -lookDir` (the local +Z axis
    // points behind the camera). Right = up × zAxis. Real up = zAxis × right.
    Vec3 lookDir = target - eye;
    const float lookLen = length(lookDir);
    if (lookLen <= 1e-6f) {
        // Degenerate: eye == target. Return identity rather than NaN.
        return Quat::identity();
    }
    lookDir = lookDir * (1.0f / lookLen);

    Vec3 upN = normalize(up);
    // If `up` is colinear with the look direction, fall back to +Z.
    if (std::fabs(dot(upN, lookDir)) > 0.9995f) {
        upN = Vec3{0.0f, 0.0f, 1.0f};
        if (std::fabs(dot(upN, lookDir)) > 0.9995f) {
            // Still colinear (caller asked for +Z look and +Z up). Use +X.
            upN = Vec3{1.0f, 0.0f, 0.0f};
        }
    }

    const Vec3 zAxis = lookDir * -1.0f;          // local +Z (behind camera)
    const Vec3 xAxis = normalize(cross(upN, zAxis));
    const Vec3 yAxis = cross(zAxis, xAxis);      // already unit; right × forward
    return quatFromBasis(xAxis, yAxis, zAxis);
}

}  // namespace iron
```

```cpp
#define IMVIEWGUIZMO_QUAT_SLERP(a, b, t)           iron::slerp(a, b, t)

// authoritative macro list). Keep this block alphabetical-ish for readability.

#define IMVIEWGUIZMO_IMPLEMENTATION
#include "ImViewGuizmo.h"

namespace iron {

void setIsometricView(FreeFlyCamera& cam, float distance) {
    // Stock 3/4 isometric: camera at (+d, +d, +d) looking toward origin.
    // The direction (-1, -1, -1)/√3 — same as Blender's NumPad-1/3/7 chord.
    cam.position = {distance, distance, distance};
    const Vec3 forward = normalize(Vec3{-1.0f, -1.0f, -1.0f});
    // FreeFlyCamera convention: yaw=0/pitch=0 looks toward world -Z.
    //   yaw   = atan2(forward.x, -forward.z) → atan2(-1, +1) = -π/4
    //   pitch = asin(-forward.y)             → asin(+1/√3) ≈ 35.26°,
    //                                          but our forward.y is -1/√3
    //                                          so pitch = -asin(1/√3).
    cam.yaw   = std::atan2(forward.x, -forward.z);
    cam.pitch = std::asin(-forward.y);
}

bool drawViewGizmo(FreeFlyCamera& cam, float size, float margin) {
    const ImVec2 viewportSize = ImGui::GetMainViewport()->Size;
    const ImVec2 gizmoPos{viewportSize.x - size - margin, margin};

    // Adapter: yaw/pitch → quat for the library, mutate, then quat → yaw/pitch.
    // Roll (euler.z) is discarded on conversion back — accepted v1 limit.
    Vec3 pos = cam.position;
    Quat rot = eulerToQuat({cam.pitch, cam.yaw, 0.0f});

    bool changed = ImViewGuizmo::Rotate(pos, rot, gizmoPos);
    if (changed) {
        cam.position = pos;
        const Vec3 e = quatToEuler(rot);
        cam.pitch = e.x;
        cam.yaw   = e.y;
    }

    // Iso button immediately below the gizmo, centered horizontally on it.
    ImGui::SetCursorScreenPos({gizmoPos.x + size * 0.25f, gizmoPos.y + size + 4.0f});
    if (ImGui::Button("Iso", ImVec2(size * 0.5f, 0.0f))) {
        setIsometricView(cam);
        changed = true;
    }
    return changed;
}

}  // namespace iron
```

**Note on `eulerToQuat`/`quatToEuler`:** these helpers (in `Quaternion.h`) take/return *degrees*, not radians. `cam.yaw` and `cam.pitch` are *radians*. Convert at the boundary:

Replace the lines:
```cpp
    Quat rot = eulerToQuat({cam.pitch, cam.yaw, 0.0f});
```
with:
```cpp
    constexpr float kRad2Deg = 180.0f / 3.14159265358979323846f;
    constexpr float kDeg2Rad = 3.14159265358979323846f / 180.0f;
    Quat rot = eulerToQuat({cam.pitch * kRad2Deg, cam.yaw * kRad2Deg, 0.0f});
```
and the writeback:
```cpp
        const Vec3 e = quatToEuler(rot);
        cam.pitch = e.x;
        cam.yaw   = e.y;
```
with:
```cpp
        const Vec3 eDeg = quatToEuler(rot);
        cam.pitch = eDeg.x * kDeg2Rad;
        cam.yaw   = eDeg.y * kDeg2Rad;
```

(Add `constexpr float kRad2Deg / kDeg2Rad` once near the top of the function or as file-scope `constexpr`.)
