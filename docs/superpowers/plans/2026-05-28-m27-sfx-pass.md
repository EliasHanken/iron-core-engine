# M27: SFX Pass + Fox-Sized Player Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `net-shooter` sound and feel like a real game. Wire 9 CC0 SFX onto existing events (gunshot, rocket-launch, 4 footstep variants, hit, jump, death scream), and right-size the player to fox dimensions (hitbox + camera height + character-controller capsule).

**Architecture:** All changes are game-side; no engine API extensions. AudioEngine's `playSoundAt` (M26) is reused for every SFX. Per-peer footstep timer + xor-shift PRNG handles 4-variant randomized cadence. Remote jump detection uses a `lastGrounded` field on `RemotePlayer` to spot the `true → false` transition. The fox-sized player is three constants (`kPlayerHalfExtents`, `kEyeHeight`, char-controller r/hh overrides) — kept on the game side so the engine's defaults stay generic.

**Tech Stack:** C++23 (MSVC `/std:c++latest`), CMake, CTest, OpenAL Soft (via M26 AudioEngine), Git LFS for *.wav. Zero new dependencies.

**Spec:** `docs/superpowers/specs/2026-05-28-m27-sfx-pass-design.md`

**Prerequisite:** M26 merged on `main`. As of this writing PR #46 is open with CI in progress; this plan assumes it lands first. Branch `feat/m27-sfx-pass` is already cut off `main` with the spec committed.

Find the `submitPlayerFox` lambda (search for `auto submitPlayerFox = `). Define a SEPARATE small lambda right after it, in the same scope (so it captures the same `audio`, `footstepSfx`, etc.):

```cpp
// M27 — emit a randomized footstep variant if the per-peer cadence elapsed.
// `state` is the fox animator's state string from submitPlayerFox's mapping.
auto maybeEmitFootstep = [&](std::uint32_t pid, const iron::Vec3& footPos,
                             std::string_view state, double nowSec) {
    if (state != "walk" && state != "run") {
        // Reset so the next step plays as soon as movement resumes.
        playerLastFootstepAtSec[pid] = nowSec;
        return;
    }
    const double cadence = (state == "run") ? 0.30 : 0.50;
    auto it = playerLastFootstepAtSec.find(pid);
    if (it == playerLastFootstepAtSec.end()) {
        playerLastFootstepAtSec[pid] = nowSec;
        return;
    }
    if (nowSec - it->second < cadence) return;

    // xorshift32 — fast non-cryptographic PRNG for variant selection.
    auto& seed = playerFootstepRng[pid];
    if (seed == 0) seed = pid * 2654435761u + 1u;  // splittable mix; never 0
    seed ^= seed << 13;
    seed ^= seed >> 17;
    seed ^= seed << 5;

    const std::size_t variant = seed % footstepSfx.size();
    audio.playSoundAt(footstepSfx[variant], footPos, /*gain=*/0.4f);
    it->second = nowSec;
};
```

Note: `state` is passed in as a `string_view` to match the `const char*` returned by `submitPlayerFox`'s state derivation. If the existing render loop has a `nowSec` already in scope (`glfwGetTime()` typically), reuse it; else add `const double nowSec = glfwGetTime();` at the top of the render loop.
