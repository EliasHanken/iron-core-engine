# M37.5 — Window resize correctness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Plumb the GLFW framebuffer-resize signal through `Window` to the sandbox host, replace all `kScreenW`/`kScreenH` compile-time-constant uses with live window dimensions, and skip rendering on a minimized (0×0) window. The Vulkan swapchain-recreate pipeline already exists end-to-end in `Renderer::setViewport`/`beginFrame` — it just needs to be called.

**Architecture:** `Window` gains a `resized_` flag and a `consumeResized()` accessor. GLFW's framebuffer-size callback updates `Window`'s live `width_`/`height_` and sets the flag (instead of discarding the dimensions, which is the current Vulkan branch behavior). The sandbox host's main-loop callback reads `window.consumeResized()` at the top of each frame: on true, calls `renderer.setViewport(w, h)` and rebuilds the projection matrix; on `w==0 || h==0`, returns early to skip the frame. Picking gets its screen size from `app.window().width()/height()` instead of compile-time constants. No unit tests — visual gate is the verification.

**Tech Stack:** C++23 (MSVC `/std:c++latest`), CMake (no presets — build dir `build-vk`), GLFW (windowing), Vulkan 1.3. Reference spec: `docs/superpowers/specs/2026-05-30-m37.5-window-resize-design.md`.

**Verification model:** No new unit tests — the resize signal depends on GLFW callbacks the OS fires; mocking is more cost than the test is worth. Every task's mechanical gate: **builds clean** (`cmake --build build-vk --config Debug --target sandbox`) and the **existing 48 tests stay green** (`ctest --test-dir build-vk -C Debug`). Final acceptance is the user visual gate (Task C1).

**Build & test commands (used by every task):**
```bash

cmake --build build-vk --config Debug --target sandbox
ctest --test-dir build-vk -C Debug --output-on-failure
```

**Branch:** already on `feat/m37.5-window-resize` (the spec commit `bf1ab2c` sits on the branch tip). Every task in this plan commits to this branch.

- [ ] **Step 2: Modify `engine/core/Window.h`**

Add the `consumeResized()` declaration in the public section after the existing `pollEvents`/`swapBuffers`/`shouldClose` accessors. Add the `resized_` field in the private section after `width_` and `height_`. To let the file-scope `framebufferSizeCallback` update private members, the simplest approach in this codebase is to add a `friend` declaration; this avoids an extra member function:

```cpp
// engine/core/Window.h — insertions only; existing members stay as-is
class Window {
public:
    // ... existing public members (width(), height(), pollEvents(), etc.)
    bool consumeResized();   // NEW: returns true once per resize event; clears the flag

private:
    // ... existing private members (handle_, width_, height_, ...)
    bool resized_ = false;   // NEW: set by framebufferSizeCallback; cleared by consumeResized

    // NEW: allow the file-scope callback to update width_/height_/resized_.
    friend void framebufferSizeCallback(GLFWwindow*, int, int);
};
```

If `framebufferSizeCallback` currently lives in an anonymous namespace, the `friend` declaration must name the namespace-qualified function. The cleanest fix: move the callback OUT of the anonymous namespace and into the `iron` namespace at file scope in `Window.cpp` (the callback name will not conflict — there's only one). The `friend` then declares it as `friend void framebufferSizeCallback(GLFWwindow*, int, int);` in the same namespace as `Window`.

- [ ] **Step 3: Modify `engine/core/Window.cpp`**

Three edits:

(a) If `glfwSetWindowUserPointer(handle_, this);` is NOT already in `Window::Window` (look just after the `glfwCreateWindow` call), add it on the line immediately after `glfwCreateWindow`. The framebuffer callback can't reach the Window instance without this.

(b) Replace the existing `framebufferSizeCallback` body with:

```cpp
// Replace the existing body. Drop the OpenGL/Vulkan #ifdef split — the Window
// state update (width/height/resized flag) applies to both backends. The
// OpenGL viewport call stays inside the OpenGL guard.
void framebufferSizeCallback(GLFWwindow* w, int width, int height) {
    auto* self = static_cast<Window*>(glfwGetWindowUserPointer(w));
    if (!self) return;
    self->width_   = width;
    self->height_  = height;
    self->resized_ = true;
#ifdef IRON_RENDER_BACKEND_OPENGL
    glViewport(0, 0, width, height);
#endif
}
```

If the callback is currently inside an anonymous namespace, move it out — either to file scope inside the `iron` namespace, or to no namespace if the existing `framebufferSizeCallback` was at no-namespace file scope. Match the rest of `Window.cpp`'s style.

(c) Add the `consumeResized()` implementation at the bottom of the file, before the closing `}` of the `iron` namespace:

```cpp
bool Window::consumeResized() {
    if (!resized_) return false;
    resized_ = false;
    return true;
}
```

```cpp
// --- M37.5: window resize + minimize guard ---
// Window updates its width/height in framebufferSizeCallback; consumeResized()
// returns true exactly once per resize and clears the flag. We mirror that
// into the renderer (which queues swapchain recreate) and rebuild the
// projection matrix's aspect ratio.
if (app.window().consumeResized()) {
    const int w = app.window().width();
    const int h = app.window().height();
    renderer.setViewport(w, h);
    if (w > 0 && h > 0) proj = computeProj();
}
// Skip the frame entirely when minimized. Returning from the setUpdate
// callback is the per-frame skip — the loop will tick again next event.
if (app.window().width() == 0 || app.window().height() == 0) {
    return;
}
```
