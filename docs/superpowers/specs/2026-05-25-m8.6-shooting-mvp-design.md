# M8.6 — Shooting MVP design

**Date:** 2026-05-25
**Milestone:** M8.6
**Status:** Design approved; implementation plan to follow.

## Goal

Ship the engine pieces a server-authoritative FPS needs (cooldowns, health,
projectile sim, sphere overlap, server-side lag compensation, clock sync) and
prove them with a small endless FFA arena shooter (`games/07-net-shooter`).
Heroes, teams, capture points, and abilities are deliberately deferred to
M8.7+.

## Scope decisions (locked during brainstorm)

| Decision         | Choice                                                                          |
| ---------------- | ------------------------------------------------------------------------------- |
| Milestone scope  | Shooting MVP only (no heroes, no teams, no capture point)                       |
| Weapon set       | Both hitscan + projectile; player switches with `1`/`2`                         |
| Lag compensation | Server rewind via `TimeHistory<Vec3>` per peer; raycast at client view-time     |
| Projectile model | Host-authoritative simulation; client renders a visual-only ghost               |
| Game loop        | Endless FFA, 2s respawn timer, kill leaderboard HUD                             |
| View             | First-person, reusing `iron::FirstPersonController`                             |
| Map              | Procedural flat square (30x30) + 4 perimeter walls + ~10 scattered cover boxes  |
| TTK              | 100 HP, hitscan 35 dmg @ 2 shots/sec, rocket 80 splash @ 0.9/sec                |
| Splash           | Linear falloff: 80 dmg at center, 0 at 4m radius                                |
| Clock sync       | Real `iron::ClockSync` helper with Ping/Pong (reserved tags 254/255)            |
| Damage broadcast | `DamageMsg` broadcast to all peers (kill feed + damage indicators for everyone) |

## Architecture

Engine helpers stay small, single-purpose, and game-agnostic. Each weapon is a
plain struct plus a pair of free functions (client-side / host-side) that
compose those helpers. No `IWeapon` polymorphic base — premature abstraction
with one consumer.

### Dependency direction

## Engine helper APIs

### `iron::WeaponCooldown` (header-only)

```cpp
namespace iron {
class WeaponCooldown {
public:
    explicit WeaponCooldown(float cooldownSec) : cooldown_(cooldownSec) {}
    bool tryFire(double nowSec) {
        if (nowSec < nextFireAt_) return false;
        nextFireAt_ = nowSec + cooldown_;
        return true;
    }
    float timeUntilReady(double nowSec) const {
        return std::max(0.0f, static_cast<float>(nextFireAt_ - nowSec));
    }
    void reset() { nextFireAt_ = 0.0; }
private:
    float cooldown_;
    double nextFireAt_ = 0.0;
};
}
```

### `iron::Health` (header-only)

```cpp
namespace iron {
struct Health { int max = 100; int current = 100; };
inline bool isAlive(const Health& h) { return h.current > 0; }
inline void applyDamage(Health& h, int dmg) {
    h.current = std::max(0, h.current - std::max(0, dmg));
}
inline void resetHealth(Health& h) { h.current = h.max; }
}
```

Damage is integer; `applyDamage` ignores negative input (no healing via
negative damage). Game decides what "dead" means — engine only exposes
`isAlive`.

### `iron::Collision`

```cpp
namespace iron {
struct RayHit { bool hit; float t; Vec3 point; Vec3 normal; };
RayHit raycastAabb(const Vec3& origin, const Vec3& dir, const Aabb& box);
bool   sphereOverlapAabb(const Vec3& center, float radius, const Aabb& box);
}
```

`raycastAabb` already exists from M4 (rope tool). Only new addition is
`sphereOverlapAabb` (closest-point-on-box distance check). If `raycastAabb`
currently lives outside `engine/game/Collision.h`, leave it; the new header
just adds the sphere helper alongside.

### `iron::ProjectileSim`

```cpp
namespace iron {
struct Projectile {
    std::uint32_t id;
    std::uint32_t ownerPeerId;
    Vec3 position;
    Vec3 velocity;
    double spawnTimeSec;
    bool alive = true;
};
struct ProjectileHit { Vec3 point; Vec3 normal; };
std::optional<ProjectileHit> tickProjectile(
    Projectile& p, float dt, std::span<const Aabb> worldBoxes);
}
```

No gravity, no player collision inside the sim. World collision only. Caller
(host) does the splash + lag-comp pass on detonation.

### `iron::LagCompensator`

Server-only. Owns one `TimeHistory<Vec3>` per peer.

```cpp
namespace iron {
class LagCompensator {
public:
    void recordPosition(std::uint32_t peerId, double timeSec, const Vec3& pos);
    void forgetPeer(std::uint32_t peerId);
    std::optional<Vec3> positionAt(std::uint32_t peerId, double rewindTimeSec) const;
    std::optional<Aabb> aabbAt(
        std::uint32_t peerId, double rewindTimeSec, const Vec3& halfExtents) const;
    struct HitscanResult {
        std::uint32_t peerId;
        float distance;
        Vec3 point;
    };
    std::optional<HitscanResult> hitscan(
        std::uint32_t shooterPeerId,
        const Vec3& origin, const Vec3& dir,
        double rewindTimeSec,
        const Vec3& playerHalfExtents,
        std::span<const std::uint32_t> candidates) const;
private:
    std::unordered_map<std::uint32_t, TimeHistory<Vec3>> perPeer_;
};
}
```

`hitscan` excludes the shooter. Rewinds every candidate peer to
`rewindTimeSec` (linear interp between samples), constructs an AABB at
`playerHalfExtents`, raycasts, returns closest hit (or `nullopt`).

### `iron::ClockSync`

```cpp
namespace iron {
class ClockSync {
public:
    void onPongReceived(double localSendTimeSec, double localRecvTimeSec,
                        double remoteTimeSec);
    bool ready() const;
    double rttSec() const;
    double offsetSec() const;   // remoteTime - localTime (rtt/2 baked in)
    double remoteTimeNow(double localNowSec) const;
private:
    static constexpr int kSamples    = 8;
    static constexpr int kMinSamples = 3;
    // ring buffer of recent samples; offset & rtt from medians
};
}
```

Implementation: ring buffer of up to 8 samples; smoothed estimates are the
medians (cheap robust statistic, handles latency spikes well).

## Wire protocol

Game-id: `kGameId = 0x4E535452u` (`"NSTR"`).
Tag 1 reserved for `peer::HelloMsg`. Tags 254/255 reserved for
`peer::PingMsg`/`peer::PongMsg` (owned by `PeerManager`, never seen by game
code). Game tags 2-10 listed below.

```cpp
namespace iron::netshooter {
constexpr std::uint32_t kGameId = 0x4E535452u;

// Client -> Host
struct PlayerInputMsg {                       // tag=2, Unreliable, ~30 Hz
    static constexpr std::uint8_t kTag = 2;
    std::uint32_t inputId;
    float dx, dy, dz;
};
struct FireHitscanMsg {                       // tag=3, Reliable, per-shot
    static constexpr std::uint8_t kTag = 3;
    float ox, oy, oz;
    float dx, dy, dz;
    double viewTimeSec;
};
struct FireRocketMsg {                        // tag=4, Reliable, per-shot
    static constexpr std::uint8_t kTag = 4;
    float ox, oy, oz;
    float dx, dy, dz;
    double viewTimeSec;
};

// Host -> Clients
struct AuthorityPositionMsg {                 // tag=5, Unreliable, ~30 Hz per peer
    static constexpr std::uint8_t kTag = 5;
    std::uint32_t peerId;
    float x, y, z;
    std::uint32_t lastInputId;
};
struct SpawnProjectileMsg {                   // tag=6, Reliable
    static constexpr std::uint8_t kTag = 6;
    std::uint32_t projectileId;
    std::uint32_t ownerPeerId;
    float x, y, z;
    float vx, vy, vz;
    double spawnTimeSec;
};
struct DespawnProjectileMsg {                 // tag=7, Reliable
    static constexpr std::uint8_t kTag = 7;
    std::uint32_t projectileId;
    float x, y, z;
};
struct DamageMsg {                            // tag=8, Reliable, broadcast to all
    static constexpr std::uint8_t kTag = 8;
    std::uint32_t attackerPeerId;
    std::uint32_t victimPeerId;
    std::uint16_t damage;
    std::uint16_t victimHpAfter;              // 0 means killed by this hit
};
struct RespawnMsg {                           // tag=9, Reliable
    static constexpr std::uint8_t kTag = 9;
    std::uint32_t peerId;
    float x, y, z;
    std::uint16_t hp;
};
struct ScoreUpdateMsg {                       // tag=10, Reliable
    static constexpr std::uint8_t kTag = 10;
    std::uint32_t peerId;
    std::uint32_t kills;
    std::uint32_t deaths;
};
}
```

### Semantic pins

- **Clock authority:** host uses its own monotonic clock directly — no
  `ClockSync` on the host side. Only clients construct a `ClockSync` (one
  per local peer, syncing against host time).
- `viewTimeSec` is the host-clock time the client was rendering when it
  fired. Client computes it as `clockSync.remoteTimeNow(now) - kInterpDelay`,
  where `kInterpDelay = 0.10` (100 ms). Host calls
  `lagComp.hitscan(..., msg.viewTimeSec, ...)` — this is the "what you
  see is what you hit" promise.
- **Player AABB half-extents** (used by lag-comp + splash):
  `kPlayerHalfExtents = Vec3{0.4, 1.0, 0.4}` (0.8m wide, 2m tall). Defined
  once in `Messages.h` so host and client agree.
- Once a `SpawnProjectileMsg` lands, clients extrapolate the ghost rocket
  locally from `position + velocity * (now - spawnTimeSec)` until a
  `DespawnProjectileMsg` arrives. No mid-flight state replication.
- Host sanity-checks `FireRocketMsg.ox/oy/oz` is within 2.0m of the
  shooter's authoritative position before spawning (anti-spoof).
- Late-joiner snapshot (host's `setOnPeerJoined` callback for `newPeer != 0`):
  send `AuthorityPositionMsg` per existing peer, `ScoreUpdateMsg` per
  scored peer, `SpawnProjectileMsg` per active rocket.

## Game-side weapons (`games/07-net-shooter`)

Each weapon = a POD struct (cooldown + tuning constants) plus a pair of free
functions.

### `HitscanRifle`

```cpp
struct HitscanRifle {
    static constexpr float kCooldownSec = 0.5f;     // 2 shots/sec
    static constexpr int   kDamage      = 35;
    static constexpr float kMaxRange    = 100.0f;
    WeaponCooldown cooldown{kCooldownSec};
};

std::optional<FireHitscanMsg> tryFireHitscanClient(
    HitscanRifle& w, double nowSec,
    const Vec3& muzzle, const Vec3& aimDir, double viewTimeSec);

struct HitscanResolution { std::optional<DamageMsg> damage; };
HitscanResolution resolveHitscanHost(
    HitscanRifle& serverWeapon, double nowSec,
    std::uint32_t shooterPeerId, const FireHitscanMsg& msg,
    const LagCompensator& lagComp,
    std::span<const Aabb> worldBoxes,
    std::span<const std::uint32_t> alivePeers,
    const Vec3& playerHalfExtents);
```

Both client and host own a `HitscanRifle`. Client cooldown gates input; host
cooldown is the authoritative rate limit (rejects spoofed bursts). Host
raycasts world boxes AND lag-compensated players, picks the closest hit. A
wall closer than the player blocks the shot.

### `RocketLauncher`

```cpp
struct RocketLauncher {
    static constexpr float kCooldownSec    = 1.1f;     // ~0.9 shots/sec
    static constexpr float kMuzzleSpeed    = 30.0f;
    static constexpr float kMaxLifetimeSec = 5.0f;
    static constexpr int   kCenterDamage   = 80;
    static constexpr float kSplashRadius   = 4.0f;
    WeaponCooldown cooldown{kCooldownSec};
};

std::optional<FireRocketMsg> tryFireRocketClient(
    RocketLauncher& w, double nowSec,
    const Vec3& muzzle, const Vec3& aimDir, double viewTimeSec);

struct RocketSpawn { SpawnProjectileMsg announce; Projectile live; };
std::optional<RocketSpawn> spawnRocketHost(
    RocketLauncher& serverWeapon, double nowSec,
    std::uint32_t shooterPeerId, const FireRocketMsg& msg,
    std::uint32_t nextProjectileId,
    const Vec3& authoritativeShooterPos,
    float maxMuzzleDistance = 2.0f);

struct RocketTickResult {
    bool expired;
    DespawnProjectileMsg despawn;
    std::vector<DamageMsg> splashHits;
};
RocketTickResult tickRocketHost(
    Projectile& live, double nowSec, float dt,
    const LagCompensator& lagComp,
    std::span<const Aabb> worldBoxes,
    std::span<const std::uint32_t> alivePeers,
    const Vec3& playerHalfExtents);

void tickRocketClient(Projectile& ghost, float dt, std::span<const Aabb> worldBoxes);
```

Splash hits every alive peer in range, **including the shooter** (rocket-jump
mechanics for free, even though we don't have jumping in the MVP). Splash
uses lag-compensated AABBs at the detonation tick's `nowSec`, not the firing
client's `viewTimeSec` — the rocket is host-authoritative from spawn forward.
Linear falloff from `kCenterDamage` at distance 0 to 0 at `kSplashRadius`.
Distance = closest-point-on-AABB to detonation point.

### Damage application chokepoint

```cpp
struct ApplyDamageOutcome { bool victimKilled; };
ApplyDamageOutcome applyDamageHost(DamageMsg& msg, Health& victimHp);
```

Both `resolveHitscanHost` and `tickRocketHost` return `DamageMsg`s with
`damage` and `attackerPeerId` filled but `victimHpAfter` at a sentinel.
`applyDamageHost` mutates the victim's HP, fills `victimHpAfter`, returns
whether the victim died. Single chokepoint for HP changes.

## Game loop wiring (`main.cpp`)

### State

```cpp
// Host-only
std::unordered_map<std::uint32_t, HostPlayer>  hostPlayers;   // hp, weapons, respawnAtSec, K/D
std::unordered_map<std::uint32_t, PlayerState> authStates;
std::vector<iron::Projectile>                  liveRockets;
std::uint32_t                                  nextProjectileId = 1;
iron::LagCompensator                           lagComp;

// Client-side
iron::PredictionEngine<PlayerInput, PlayerState> predictor;
std::unordered_map<std::uint32_t, RemotePlayer>  remotes;        // TimeHistory + HP for HUD
std::unordered_map<std::uint32_t, iron::Projectile> ghostRockets;
LookState                                        look;
HitscanRifle                                     localHitscan;   // client-side cooldown gate
RocketLauncher                                   localRocket;    // client-side cooldown gate
int                                              selectedWeapon = 1;
int                                              localHpForHud  = 100;
std::vector<KillFeedEntry>                       killFeed;       // capped to 5
```

### Three fixed-step accumulators (built on `iron::FixedTickScheduler`)

- `inputTicker` @ 30 Hz — client samples input + sends `PlayerInputMsg` +
  predictor.applyInput. Host does the same locally for its own player.
- `simTicker` @ 60 Hz — host-only. Apply queued inputs to authStates;
  record `lagComp.recordPosition`; tick rockets; process respawn timers;
  broadcast `AuthorityPositionMsg`.
- `renderTicker` @ vsync — all peers. Render local from predictor (or
  authStates[0] on host), remotes from `TimeHistory` at
  `nowSec - kInterpDelay`, ghosts from extrapolation.

### Pre-Hello reconcile guard

Same lesson as net-tag's M8.5 fix: the
`AuthorityPositionMsg` handler checks `!peers.hasIdentity()` and routes
early messages to `remotes[].positionHistory` instead of calling
`predictor.reconcile()`. Without this, a pre-Hello client (myPeerId == 0)
collides with the host's peerId 0 and yanks its own predictor to the host's
position.

### Late-joiner snapshot

Inside `peers.setOnPeerJoined([&](pid){ ... })`, when host sees `pid != 0`,
broadcast to `pid`:
- `AuthorityPositionMsg` per existing peer (skipping `pid`),
- `ScoreUpdateMsg` per peer with non-zero kills/deaths,
- `SpawnProjectileMsg` per entry in `liveRockets`.

### Death / respawn

When `applyDamageHost` returns `victimKilled`:
- `victim.deaths++`, `attacker.kills++`
- `victim.respawnAtSec = nowSec() + 2.0`
- Broadcast `ScoreUpdateMsg` for both peers
- Host stops applying victim's `PlayerInputMsg`s and stops broadcasting
  `AuthorityPositionMsg` for them; `lagComp` stops recording until respawn

Each sim tick scans `hostPlayers` for `!isAlive && nowSec >= respawnAtSec`,
resets HP, picks a random spawn, broadcasts `RespawnMsg`.

### Input map

- `WASD` / mouse: movement + look (FirstPersonController-style)
- `1` / `2`: select hitscan / rocket
- `LMB`: fire the selected weapon
- `ESC`: release cursor

### HUD (`iron::Hud`)

- Top-left: kill leaderboard (peerId : K/D), local player highlighted
- Top-right: net-stats widget (M8.4)
- Center: crosshair
- Center-low: selected-weapon label ("RIFLE" / "ROCKET")
- Bottom-left: HP bar (0..100) + numeric
- Bottom-right: kill feed (last 5: "A → B")
- Death overlay: "Respawning in 2.0s..." countdown

## Arena

```cpp
struct Arena {
    std::vector<iron::Aabb> boxes;       // floor + 4 walls + ~10 cover boxes
    std::vector<Vec3>       spawnPoints; // 4 corners
    Vec3 randomSpawn() const;
};
```

30x30 floor at y=0. 4 perimeter walls. 10 cover boxes (size ~2x2x2)
placed on a fixed grid with seeded per-cell offsets so host and clients
generate identical geometry. 4 spawn points at the corners. RNG seeded by a
constant — no networked seed sync needed because clients render the arena
locally and don't shoot against it.

## Risks / open questions

- **Clock sync convergence.** First 1-2 seconds after Hello, clients have
  `clockSync.ready() == false` and can't compute `viewTimeSec`. Plan: client
  blocks firing until `clockSync.ready()` is true; HUD shows "Syncing..." if
  > 3s.
- **TimeHistory memory.** Recording every peer's position at 60Hz for a
  reasonable rewind window (~250ms) is small (~15 samples), but if a server
  runs long we should ensure `TimeHistory` prunes old samples. Verify
  during M8.6 implementation; if not pruning, extend it.
- **Player AABB collision against world.** MVP has no player-vs-wall
  collision — players can walk through walls. Acceptable for the demo
  (arena is open enough that this isn't immediately obvious). Real world
  collision is a future task; flagged for follow-up.

## Out of scope (deferred)

- Heroes / archetypes (M8.7)
- Teams + capture point (M8.7 or M8.8)
- Abilities (M8.8)
- Player-vs-world collision (engine follow-up)
- Player-vs-player collision (deliberately none — players phase through
  each other in MVP, simplifies movement code)
- Audio (engine has no audio subsystem yet)
- Spectator mode for dead players (currently you just see your last position
  frozen for 2s)
