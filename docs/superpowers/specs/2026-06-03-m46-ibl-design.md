# M46 — Image-Based Lighting (IBL) Design

**Date:** 2026-06-03
**Track:** Render-polish (realism-first). Sits between M45 PBR (done) and M47 bloom.
**Status:** Approved design. Implemented as three PRs — M46a, M46b, M46c.

## Problem

The standard lit shader (`engine/render/StandardLitShader.h`, fragment source
~lines 117–266) currently fakes the environment two ways:

- **Diffuse ambient** = a flat constant color: `vec3 ambient = u.ambient.xyz * albedo * ao;`
  (same intensity from every direction).
- **Metal reflection** = a raw `reflect(viewDir, N)` lookup into the skybox cubemap
  (binding 5), **not** weighted by roughness or Fresnel. This M16-era "crude mix"
  is why a metal cylinder reflects the raw sky and reads as "not metallic."

Industry-standard **split-sum IBL** (Karis/UE4) replaces both with three resources
precomputed from the environment:

1. **Irradiance cubemap** — small (32×32), cosine-convolved → direction-varying diffuse.
2. **Prefiltered specular cubemap** — a mip chain where mip *m* is the environment
   blurred for roughness `m/(mips-1)`.
3. **BRDF integration LUT** — a 2D `RG16F` texture, `f(NdotV, roughness)` → the
   scale/bias term. Scene-independent; baked once.

Final ambient becomes:
`ambient = (kD · irradiance·albedo + prefiltered·(F0·brdf.x + brdf.y)) · ao`.

## Existing foundations (all present)

- `VkComputePipeline` — proven compute dispatch (see `VkParticleSystem.cpp`).
- `VkCubemap` / `VkCubemapStore` — 6-layer cube-compatible images, shared sampler
  (LINEAR/LINEAR mip, CLAMP_TO_EDGE). Currently `RGBA8_SRGB` only.
- Runtime GLSL→SPIR-V via glslang (`VkShader.cpp::compileGlsl`), supports compute.
- `stb_image` vendored (`third_party/stb/`); `stbi_loadf` decodes Radiance `.hdr` for free.
- Skybox path: `createCubemap(w,h, 6 RGBA8 faces)` → `setSkybox(handle)`
  (`Renderer.h:146–152`); `VkSkybox` renders it; `ProceduralSky.cpp` generates a
  sunset-gradient LDR cube.
- Lit descriptor set: bindings 0–8 + bones UBO at 9 (skinned). `LitUbo` = 960 B.

This makes IBL **purely additive** to proven patterns.

## Milestone breakdown (three PRs)

The equirect→cube conversion is itself the first compute-bake, so M46a bootstraps the
bake infrastructure (`IblBaker`) that b and c reuse. Each PR is independently testable,
visually gateable, and ships the engine in a working state.

| PR | Scope | Visual gate |
|----|-------|-------------|
| **M46a** | HDR environment pipeline: `.hdr` equirect loader (`stbi_loadf`) + equirect→cubemap compute + `RGBA16F` HDR cubemap path + `loadHdrSkybox()` API + `IblBaker` foundation. | Real HDR environment visible in skybox; existing crude reflection picks it up. |
| **M46b** | Diffuse IBL: irradiance cubemap bake (cosine convolution compute) + diffuse-ambient rewire (binding 10). Replaces flat `u.ambient`. | Ambient becomes directional — objects lit by environment color, not a flat constant. |
| **M46c** | Specular IBL: prefiltered specular mip chain + BRDF integration LUT + specular rewire (bindings 11, 12). Replaces M16 crude reflection. | Metal cylinder finally looks metallic; roughness drives reflection blur. |

## Architecture & components

### `IblBaker` (`engine/render/backends/vulkan/VkIblBaker.{h,cpp}`)
Owns the "compute writes into a cubemap" pattern shared by all three bakes:
- Creates an HDR cube target (`RGBA16F`, `STORAGE | SAMPLED`, cube-compatible) with N mips.
- Exposes a **2D-array storage view** (all 6 layers) per mip for `imageStore` from
  compute, plus a **cube sampled view** for the lit pass.
- `dispatch(cb, pipeline, set, faceSize)` →
  `vkCmdDispatch(ceil(faceSize/8), ceil(faceSize/8), 6)` (X,Y = face texels, Z = 6 faces),
  wrapping the layout transitions (`GENERAL` for write → `SHADER_READ_ONLY` for sample).

### `VkCubemapStore` HDR variant
Add `createHdr(width, height, mips, usage)` → `RGBA16F` cube-compatible resource.
Used by the equirect→cube pass (M46a) and the irradiance/prefilter targets (b/c).
Existing `createFromFaces` (`RGBA8_SRGB`) stays for LDR/procedural skyboxes.

### Renderer API (Vulkan-only; OpenGL stubs per the vulkan-only direction)
- M46a: `CubemapHandle loadHdrSkybox(const std::string& hdrPath, int faceSize)` —
  loads `.hdr`, runs equirect→cube, returns an HDR cubemap handle usable with the
  existing `setSkybox()`.
- Irradiance / prefilter / BRDF-LUT resources are **internal** — auto-derived from
  whatever cubemap is set as skybox, not separate public handles.

### Compute shaders (runtime-compiled GLSL via glslang)
1. `equirectToCube.comp` (M46a) — sample equirect `sampler2D` by per-texel cube
   direction → `imageStore` into the face layer.
2. `irradianceConvolve.comp` (M46b) — hemisphere cosine-weighted integral of the env
   cube → 32×32 irradiance cube.
3. `prefilterEnv.comp` (M46c) — GGX importance-sampled prefilter; one dispatch per mip
   with `roughness = mip/(mipCount-1)` → prefiltered cube (128×128, 5 mips).
4. `brdfLut.comp` (M46c) — split-sum scale/bias integration → 512×512 `RG16F` 2D
   storage image. Baked once at startup (environment-independent).

## Lifecycle & fallback

**Bake-once, not per-frame.** IBL resources are static for a given environment. On
`setSkybox`, set `iblDirty_`. At the next frame start (before the lit pass), if dirty:
run the bake passes once, clear the flag. Re-bakes only on skybox change.

**Fallback when no skybox.** Several games run without one. The lit shader carries an
`iblEnabled` flag (UBO slot); when 0 it uses today's flat `u.ambient` path, so
non-skybox scenes render exactly as they do now. IBL only activates when an
environment is present. The procedural sunset sky still works as an IBL source (LDR
range, but valid); `loadHdrSkybox` provides true HDR.

## Shader rewire (split-sum) & bindings

**Descriptor bindings grow 9 → 12.** To stop reshuffling the bones UBO every milestone
(it moved 8→9 in M45c), give bones a **fixed high binding** (15) now so future sampler
growth never collides with it again.

| Binding | Resource | Added |
|---|---|---|
| 0 | `LitUbo` | existing |
| 1–4 | baseColor / normal / metallic-rough / shadow | existing |
| 5 | skybox cube (still used by `VkSkybox` + LDR fallback) | existing |
| 6–8 | planar reflection / AO / emissive | existing |
| 10 | **irradiance cube** (`samplerCube`) | M46b |
| 11 | **prefiltered specular cube** (`samplerCube`) | M46c |
| 12 | **BRDF LUT** (`sampler2D`) | M46c |
| 15 | bones UBO (skinned only) — fixed high binding | moves here |

Fragment math replacing `StandardLitShader.h` ambient/reflection (~lines 242–257):

```glsl
// diffuse (M46b)
vec3 irradiance = texture(uIrradiance, N).rgb;
vec3 diffuseIBL = irradiance * albedo;

// specular (M46c) — split-sum
vec3  R          = reflect(-V, N);
float maxMip     = float(textureQueryLevels(uPrefiltered) - 1);
vec3  prefiltered = textureLod(uPrefiltered, R, roughness * maxMip).rgb;
vec2  brdf       = texture(uBrdfLut, vec2(max(dot(N, V), 0.0), roughness)).rg;
vec3  F          = fresnelSchlickRoughness(max(dot(N, V), 0.0), F0, roughness);
vec3  specularIBL = prefiltered * (F * brdf.x + brdf.y);

vec3 kD      = (1.0 - F) * (1.0 - metallic);
vec3 ambient = (kD * diffuseIBL + specularIBL) * ao;   // replaces flat u.ambient
```

When `iblEnabled == 0`: `ambient = u.ambient.xyz * albedo * ao` (today's path). M46b
lands only the diffuse half (specular stays crude until M46c).

**`LitUbo` (960 B → 976 B):** add IBL controls in spare slots —
`materialParams2.w = iblEnabled`, and a new `vec4 iblParams{ intensity, maxReflectionLod, _, _ }`.
16-byte alignment preserved.

## Testing

Following the `test_pbr` / `test_tonemap` convention (CPU port of shader math,
standalone binary, `approx()` asserts):

**`test_ibl`** validates:
- Cube-direction reconstruction per face round-trips equirect↔direction.
- Cosine-weighted irradiance integral of a constant environment ≈ the input
  (energy conservation).
- GGX-prefilter at roughness 0 ≈ a mirror sample.
- BRDF LUT endpoints: at roughness 0, NdotV 1 → scale ≈ 1, bias ≈ 0.

**Visual gates** per PR (table above) are the real validation for the bake passes —
DamagedHelmet + a metal cylinder under a known HDR environment.

## Risks / edge cases

- **Compute can't write sRGB storage images** → all bake targets are `RGBA16F` (linear).
- **Cubemap seams** at low mips → `CLAMP_TO_EDGE` + cube-compatible images; seamless
  cube filtering is core in Vulkan.
- **Layout transitions** (compute-write → fragment-sample) are the main correctness
  risk → centralized in `IblBaker::dispatch`.
- **glslang compute support** already proven (`VkParticleSystem`).
- **Bones-UBO rebind** to a fixed high binding touches both skinned shader source and
  the skinned descriptor-layout setup — code-review per the always-review rule.

## Out of scope (future milestones)
- M47 bloom, M48 SSAO, M49 displacement (per render-polish roadmap).
- Multiple/blended environment probes; runtime dynamic-environment re-bake throttling.
- Specular occlusion / horizon fading refinements.
